# Manual regression (Windows-first)

## Самопроверка визуализации
- Запуск `incline3d --render-selftest <путь>` на Windows (MSYS2/GTK4), проверить наличие `axonometry.png`, `plan.png`, `vertical.png`.
- Проверить экспорт путей с Unicode/пробелами/русскими символами.
- Размеры изображений: 1024×768, файлы ненулевые.

## UI (Windows)
- Открытие проекта, сохранение с путём Unicode (NTFS, регистронезависимость).
- Масштабирование HiDPI (125%/150%): читаемость сеток/подписей.
- Управление мышью:
  - 3D: ЛКМ — вращение; Ctrl+ЛКМ — панорама; колесо/Shift+ЛКМ — масштаб.
  - План/Вертикальная: колесо — масштаб, перетаскивание — панорама.
- Цвета/освещение: нет инверсии/битых шрифтов в GDK Win32 backend.
- Буфер обмена/экспорт PNG (если реализован) — корректные размеры, без артефактов.

## Алгоритмы/данные
- Режимы азимутов: Auto/Magnetic/True; интерполяция пропусков и extend-last.
- Blanking азимутов на вертикали: X/Y не растут, INT10/INT_L = 0.
- Dogleg метод: переключение Sine/Cosine даёт ожидаемое изменение интенсивности.

## Печать/отчёты (если добавлены)
- Папки назначения с пробелами и русскими буквами.
- CRLF в экспортируемых текстовых файлах.
- CLI `--diagnostics --out <путь>`: наличие report.md/json, корректные статусы, изображения в `images/`.
- CLI `--report-analyses --out <путь>`: Markdown+CSV созданы, числа детерминированы.
- Диалог «Диагностика…»: запуск не блокирует UI, путь к отчёту открывается, статусы соответствуют CLI.
- Диалог «Анализы…»: выбор базовой/целевой скважины, генерация отчёта, корректная смена пути при повторном запуске.

## Импорт CSV/LAS (Windows-first)
- CLI: `incline3d --import-measurements tests/fixtures/Incl.csv --format csv` — успешно, >200 точек, глубины монотонны, углы в [0;180].
- CLI: `incline3d --import-measurements tests/fixtures/gir1.las --format las` — успешно, >1000 точек, глубины монотонны.
- Проверить ручной маппинг:
  - CSV: `--depth-col 1 --inc-col 2 --delimiter ';' --encoding CP1251` импортирует Incl.csv без ошибок.
  - LAS: `--depth-mnemonic DEPTH --inc-mnemonic ZENIT` импортирует gir1.las без ошибок.
- UI (когда доступен импорт): загрузка этих же файлов через диалог, корректное определение колонок/кривых, понятная ошибка при намеренном переименовании кривых.
